  <section class="space-y-6 max-w-3xl mx-auto">
    <h2 class="text-2xl font-semibold text-neutral-900 dark:text-neutral-100">What is @layer?</h2>
    <p class="text-neutral-600 dark:text-neutral-400 leading-relaxed">
      The <code class="text-purple-600 dark:text-purple-300 bg-purple-50 dark:bg-purple-900/50 px-1.5 py-0.5 rounded">@layer</code> at-rule lets you organize CSS into named cascade layers. Layers have explicit precedence: the first declared layer has the lowest priority, and the last declared layer has the highest. Un-layered styles always override layered styles.
    </p>
    <p class="text-neutral-600 dark:text-neutral-400 leading-relaxed">
      The key insight: once layer order is established, the cascade ignores selector specificity within layers. A rule in a later layer wins over a rule in an earlier layer—even if the earlier rule has a more specific selector. This lets you use simpler selectors and avoid specificity hacks.
    </p>

    <h2 class="text-2xl font-semibold text-neutral-900 dark:text-neutral-100">!important vs @layer</h2>
    <p class="text-neutral-600 dark:text-neutral-400 leading-relaxed">
      With <code class="text-purple-600 dark:text-purple-300 bg-purple-50 dark:bg-purple-900/50 px-1.5 py-0.5 rounded">!important</code>, you create an override war. To override a style that uses <code>!important</code>, you need another <code>!important</code>. This leads to specificity escalation and CSS that becomes hard to maintain—especially when mixing third-party styles, design systems, and utilities.
    </p>
    <p class="text-neutral-600 dark:text-neutral-400 leading-relaxed">
      With <code class="text-purple-600 dark:text-purple-300 bg-purple-50 dark:bg-purple-900/50 px-1.5 py-0.5 rounded">@layer</code>, you define layer order once (e.g. <code>@layer base, components, utilities;</code>). Later layers override earlier layers without any <code>!important</code>. It's the modern, maintainable way to control precedence when working with multiple CSS sources.
    </p>
    <p class="text-neutral-600 dark:text-neutral-400 leading-relaxed">
      Note: when <code>!important</code> is used, layer priority reverses—the first declared layer with <code>!important</code> wins. This can be useful when you want a base layer to "win back" over overrides in certain cases.
    </p>
  </section>
